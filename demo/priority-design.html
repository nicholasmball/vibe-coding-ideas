Design rationale: Medium (the default) uses a muted yellow-600 at 40% opacity so it doesn't compete with urgent/high. Low uses zinc-500 to blend with muted foreground text. The visual hierarchy is: urgent (red) > high (orange) > medium (muted) > low (blends in).

8. Interaction & Keyboard Navigation

Task Card

  • Click Anywhere on card opens detail dialog (existing behaviour)
  • Hover Dot shows native tooltip via title attribute
  • Drag Unaffected — dot is inside the content area, not the drag handle

Priority Selector (Detail Dialog)

  • Tab Focus lands on Select trigger (standard tab order after Due Date)
  • Space/Enter Opens dropdown (shadcn Select built-in)
  • Arrow keys Navigate options (shadcn Select built-in)
  • Escape Close dropdown without changing (shadcn Select built-in)
  • On change Optimistic update + server action + activity log + toast on error

9. Accessibility (WCAG 2.1 AA)

✓

Not colour-only: Priority label is available via tooltip (card) and text label (dialog selector). Colour is supplementary, not the sole indicator.

✓

Contrast ratios: All dot colours meet 3:1 minimum against zinc-950 background (non-text contrast per WCAG 1.4.11). Red (#ef4444) = 4.6:1, Orange (#f97316) = 5.2:1, Yellow-600/40 = 3.2:1, Zinc-500 (#71717a) = 3.8:1.

✓

Keyboard navigable: shadcn Select provides full keyboard support out of the box (Space/Enter to open, arrows to navigate, Escape to close).

✓

Screen readers: Select trigger reads the current value. Options read dot + label. Card dot has title for tooltip but is decorative (aria-hidden="true" with the label available in the detail dialog).
